## 問題のあるページ

### 問題1: 銘柄詳細 — 主要指標が全て `-`

- **ページ**: `/stock/{code}` (例: `/stock/7203`)
- **症状**: 配当利回り、PER、PBR、ROE、自己資本比率、配当性向、連続非減配、連続増配、5年配当CAGR が全て `-`
- **原因**: フロントは `getDividendCheck()` レスポンスの `raw` フィールドを参照 (`div.raw.per`, `div.raw.div_yield` 等)。リファクタリング後の `/api/dividend/check/{code}` は `raw` を返していない
- **データソース**: `stock_fundamentals` テーブルに値はある
- **対象ファイル**: `findex/api/routers/dividend.py` の `dividend_check()`

### 問題2: モメンタムランキング — 騰落率列が全て `-`

- **ページ**: `/momentum`
- **症状**: 12M騰落、12M相対、3M騰落、3M相対 の4列が全てハイフン
- **原因**: フロントは `ret_12m`, `ret_3m`, `rel_ret_12m`, `rel_ret_3m` を参照するが、APIはスコア値（0〜10点の `breakdown`）のみ返しており、実数値（騰落率%）を返していない
- **データソース**: `price_history` から計算が必要（`momentum_scores` テーブルには実数値カラムなし）
- **対象ファイル**: `findex/api/routers/momentum.py` の `momentum_rank()`

### 問題3: モメンタム詳細 — `fields` がない

- **ページ**: `/stock/{code}` のモメンタムタブ
- **症状**: 12Mリターン、3Mリターン、52週高値比率、売上成長率、EPS成長率が全て `-`
- **原因**: フロントは `mom.fields.ret_12m` 等を参照するが、APIは `fields` フィールドを返していない
- **データソース**: `price_history` + `stock_fundamentals` から計算が必要
- **対象ファイル**: `findex/api/routers/momentum.py` の `momentum_check()`

| 問題 | ステータス | 根拠 |
|---|---|---|
| 問題1: 銘柄詳細の主要指標 `-` | ✅ 解消済み | `dividend_check()` に `raw` フィールド追加。実レスポンス確認済み（div_yield, per, roe, equity_ratio 等全て返却） |

### 問題1 の詳細

新しい `findex/api/routers/dividend.py` の `dividend_check()` は以下のキーで `raw` を構築:

```
div_yield, per, pbr, mix_coefficient, net_cash_per,
equity_ratio, debt_to_equity, roe, operating_margin(※),
payout_ratio, eps_growth_5y, revenue_growth_5y_cagr,
consecutive_no_cut_years, consecutive_dividend_growth_years,
dividend_growth_5y_cagr, dividend_growth_10y_cagr,
dividend_reliability, dividend_cut_count_20y,
fcf_payout_coverage, roic_minus_wacc,
retained_earnings_div_ratio, market_cap
```

フロントの `StockDetail.tsx` が参照するキー:
```
div_yield, per, pbr, roe, equity_ratio, payout_ratio,
consecutive_no_cut_years, consecutive_dividend_growth_years,
dividend_growth_5y_cagr, dividend_cut_count_20y, market_cap
```

→ **全て含まれている。** ただし `roe` のソースが `raw_financials.roe`（yfinanceの `returnOnEquity` そのまま）であり、`stock_fundamentals.roe` と同一値なので問題ないはず。

**残アクション**: `curl localhost:8080/api/dividend/check/7203` で `raw` フィールドの値が null でないことを確認する。null なら `raw_financials` にデータが入っていないことが原因。
