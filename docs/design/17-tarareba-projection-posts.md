# 17 - タラレバ（前提付き配当試算）を全テーマの文章2〜4へ

> 起点: posts.html の投稿パターンを増やす施策。「もし長期保有したら配当はこう育つ」という *タラレバ*
> （前提付き試算）の切り口を、全テーマに第2〜第4の本文案として持たせる。
>
> 関連: [[16-post-body-top3-names]]（文章構成・配当併記・140字制限） [[04_5-indicator-calibration]]

## 1. 最終形（ユーザー方針 2026-06-21）

各テーマは **1つの画像に対し複数の本文案** を持つ:

- **文章1（説明）**: そのテーマの看板を説明するランキング本文（doc16）。
- **文章2〜4（タラレバ）**: 同じテーマの画像内TOP銘柄を主役にした3つの前提付き試算:
  1. **将来の配当** — 現在 / 10年後 / 20年後の年配当（100万円投資時）。
  2. **月3万円の元手** — 月3万円(年36万円)の配当に必要な元手（今すぐ vs 10年待ち）。
  3. **配当で元本回収** — 受取配当の累計が投資元本100万円に達する年数。
- **画像**: テーマのランキング表（全文章で共通）。

投稿運用: ギャラリー（posts.html）で文章ごとに「本文をコピー」＋字数バッジを並べ、ユーザーが
1案を選んで画像と共に投稿する。**専用タラレバ3テーマ（旧 future_dividend / road_to_3man /
dividend_doubling）は本機構へ統合し廃止**（「文章1＋文章2〜」の統一構成に一本化）。

## 2. 試算ロジック（`_tarareba_calc`・確証データのみ）

- 入力は status=ok の `dy`（利回り）/ `dpc5`（増配率5年）/ `eps_growth_5y` / `payout_ratio` のみ。
  どれか欠ければ **None＝対象外**（確証なき数字は出さない＝定款）。確定claimでなく **前提付き試算**。
- 3シナリオ: **慎重**(初期≤15%→15年で4%へ逓減) / **標準**(増配率上限10%) / **強気**(増配率5年を維持)。
  全シナリオ共通で **配当性向が cap(70%) に達したら増配率を EPS成長率へ減速**（利益を超える配当を作らない）。
- 投稿本文に出すのは原則 **標準シナリオ**。本文に「増配力が続く試算なら」と条件を明示。

## 3. 主役の選び方（本文⇔画像の整合）

`_tarareba_for(shown)`:
- そのテーマの **画像内TOP5（`shown[:5]`）** から、`_spot_eligible`（薄データ/タコ足除外・連続増配≥3年で
  無配復配のYoCジャンプを排除）かつ `_tarareba_calc` が成立する銘柄を候補にする。
- **3角度それぞれで「候補内の最良銘柄」を主役**にする（将来配当=20年後配当 / 月3万=10年後YoC /
  元本回収=回収年数）。主役は必ず画像の表に載る＝本文と画像が矛盾しない。
- 候補が無いテーマは **タラレバを出さない（文章1のみ）**。例: roic_spread や value/cash 系で、TOP銘柄が
  増配株でない場合。データが支えない切り口を捏造しない（定款）。データ集合（cohort/all）で変動する。

## 4. 実装（findex/post/themes.py）

- `_lead_future` / `_lead_3man` / `_lead_payback`: 各角度の本文（モジュール関数）。`_TARAREBA_ANGLES` に
  (ラベル, lead, 角度内の最良選定キー, 降順) で束ねる。
- `_tarareba_for(shown)` → `[{label, body}, ...]`。各テーマのビルダー戻り値に `"tarareba"` を追加
  （`_ranking_theme` ＋ カスタム5本 streak/high_yield_safe/div_growth/value_quality/net_cash）。
- ギャラリー: `_post_block(kind, body, label)` が見出し（説明／タラレバ：ラベル）＋コピー＋字数バッジ＋
  本文を組む。1テーマ＝文章1＋タラレバ各案を縦に並べ、画像（`cardwrap`）は最後に1枚共通。

## 5. 制約・正直性

- **全文章 ≤ 実140字**（[[16-post-body-top3-names]] §7。`post_len` 判定）。タラレバ本文は実測 117〜131字。
- 試算は「試算」と明示（本文の語＋字面）。確定値（利回り等）と前提付き試算を混同させない。
- 主役は画像内TOP5限定＝本文の銘柄が必ず画像に出る。タラレバ不能テーマは沈黙（出さない）。

## 6. 追補（同日）: タブUX＋締めのテーマ固有化

- **タブUX**: 1テーマの複数文章案（説明／タラレバ各角度）を縦積みすると長大になるため、ギャラリーは
  **タブ切替**にした（`_post_tab` がタブ＋パネル対を返す。CSS `.tab/.panel.active`、JSでパネル切替）。
  初期表示は「説明」。画像（`cardwrap`）はタブ外＝全案で共通表示。
- **締めのテーマ固有化**: 旧実装は3角度すべて「増配力が続けば…」の汎用煽りで、テーマをまたぐと単調・
  違和感。`_TARAREBA_THESIS`（テーマ→看板の論理の一文）を lead に伝播し、締めをテーマ固有の根拠に。
  例: high_yield「高い利回りを今のうちに。」/ doe_king「資本に報いるDOEが裏づけ。」/
  high_roe_growth「高いROEが増配を生む。」/ growth_room「低い配当性向＝増配余地。」。
  ＝テーマの選別理由で試算の現実味（理論の正しさ）を裏づける。

- **タラレバ①の締めを「配当の倍率」→「実質利回り(YoC)20年後」へ**（2026-06-21・バグ修正）:
  旧実装の締め `約{div[(base,20)]/init:.1f}倍` は、標準シナリオの増配率上限が一律10%（`_SCN_CEILING`）
  ＝ほぼ全銘柄が `1.10^20≒6.7倍` に潰れて**定数化**（cohortで32本中28本が「6.7倍」・テンプレに見えた）。
  倍率は `init`（=投資額×dy）が分母分子で相殺し銘柄差を消す。締めを **YoC20年後（=倍率×dy）** に
  変えると現在利回りに比例して銘柄差が出る（cohort 9.4〜26.9%・all 16〜65%）。「将来の配当」の看板
  KPIとしても自然。`_body_metric(c['yoc'][('base',20)],'pct')`。回帰テスト
  `test_lead_future_headline_varies_by_stock`。本文短縮（「現在の利回り：X%（年Y円）」の年Y円を削除）で
  全文 ≤140字を維持。
  - **表記は「実質利回り」**（当初「買値利回り」だが直感性で改称・同字数）。「実質」は税引後/インフレ
    調整でなく**買値基準(YoC)**の意。誤認回避のため「試算」「20年後」の文脈に必ず乗せる（本文上部に
    「🌳20年後：年◯円」が並び末尾も「試算では…」始まり＝現在の確定利回りと取り違えない構成）。

## 6.5 説明本文の改行整形（読みやすさ・2026-06-21）

タラレバ本文は段落間が空行で読みやすい一方、文章1（説明）はフック／ランキング見出し＋メダル列／
締めが詰まって読みにくかった。**説明本文も段落間に空行**を入れる:

```
フック1（引き）
            ← 空行
〇〇ランキング📈 トップN
🥇… 配当X% …
🥈…
🥉…
            ← 空行
締めフック＋#タグ
```

- 実装は**中央化**（17テーマのlambdaを個別編集しない）: メダル列〜締めの空行は `_post_name_block` の
  末尾を `\n\n` に、フック1直後の空行は `_lead_break(body)`（先頭行の後に空行を挿入）を
  `_ranking_theme` ＋ カスタム5本（streak/high_yield_safe/div_growth/value_quality/net_cash）に適用。
- 空行で実文字数 +2字／テーマ。最長135字で **≤140字** を維持（`--cohort`/`--all` 両方確認）。
