# 19 - 流動性フロア（テーマ層・発信事故の足切り）

> 起点: ユーザー指摘「100億円以下の株は流動性が低い気がする。スコアリングに影響しているか」（2026-06-24）。

## 背景・問題
柱2の採点エンジン（`findex/score/engine.py`）は時価総額を**点数に使っていない**（中立）。
時価総額の唯一の用途は `select_rules` の分類スイッチ（large_cap_threshold=1兆円で代替指標へ
入れ替え）だけで、出来高・売買代金などの流動性指標はDBにもスコアにも存在しない。

結果、薄商いの小型株が利回り等で上位に来ても止める仕組みがなく、そのまま投稿候補になる。
実データ（2026-06-24・全3,673社中 時価総額<100億=1,252社=34.1%）で確認した生利回りテーマの
現状トップ10は **4/10 が100億円未満**、しかも**1位がトーシンHD＝時価総額13億円**。
「気配が薄くて実際には買えない株」を高配当ランキング首位として発信する事故になっていた。

## 決定（案1: テーマ層フロア）
- **柱2スコアは中立のまま**（小型株も同じルールで採点）。時価総額で点数を歪めない。
- **発信層（柱3）でのみ足切り**: 時価総額 **100億円（`MIN_MARKET_CAP=1e10`）未満**を
  全テーマの投稿候補から除外する。
- 実装は `findex/post/themes.py` の `_liquid(r)`。既存の薄データゲート `_sufficient(r)` と
  同じ全テーマ共通地点（宣言的テーマは `_ranking_theme`、手書きビルダ計7箇所）に併置。
- **`current_market_cap` 未取得(None)は除外しない**＝「確証なき除外もしない」（定款）。
  タコ足(EPS5=None)・業種フィルタ(sector None)と同じ非除外扱い。

## 効果（実測・2026-06-24）
- 生利回りテーマ eligible: 850 → 637（**213社除外**）。トップ10からトーシンHD(13億)・
  ベリテ(59億)・Hamee(43億)・シライ電子(68億)が落ち、買える銘柄が繰り上がる。
- `small_value`（時価総額<1000億）テーマは 100億〜1000億 に縮むだけで主旨は温存。

## 限界・将来
時価総額は流動性の**近似**。一株が大型でも薄商いはあり得る。より厳密には
売買代金（`price_history` の volume×終値の中央値）でのフロアが理想で、これは新規取得なしに
derive 可能（鉄則: 新規フェッチは避ける）。まずは素直な proxy で発信事故を止め、
売買代金フロアは精緻化候補として保留。
