# D8: バックテスト基盤（モデル検証）

**作成日**: 2026-06-15
**親**: [charter](00-charter-and-data-integrity.md)（柱2＝進化する指標）/ [D4 指標システム](04-indicator-system.md) / [D4.5 較正](04_5-indicator-calibration.md) / [design-review](design-review.md) #1
**目的**: **総合スコア・claim別グレード・各指標が「実際に望ましい結果を予測するか」を実証する。** データの正確性（柱1）と同じ厳密さを、**モデルそのもの**に適用する。
**核心原則（定款の拡張）**: 「確証を持てない数字を流さない」を**モデルにも適用**する。重み（2.5/1.5/1.2…）は旧PJ継承の手づけであり、予測力の確証が無い。**確証無きモデルは確証無きデータと同じく有害**。バックテストはその確証を与える装置。

---

## 0. なぜ必要か（最大の盲点の解消）

- 入力データは D2〜D6 で厳密に検証される。だが**スコアの重み配分・指標選定は一度も予測力を検証されていない**（D4.5は閾値をパーセンタイル化したが重みは未検証）。
- 「100%正確なデータ ＋ 恣意的なモデル ＝ 自信ありげな無意味」。これは定款が憎む有害物と構造的に同じ。
- 「進化する指標」（柱2）には**進化の方法論**が要る。何を信号に重みを直すのか＝**バックテストのアウトカム**。本書がそれを定義する。

---

## 1. 設計の二大課題

### 1.1 PIT（Point-in-Time）正確性 — 先読みバイアスの排除
「2018年時点のスコア」は、**2018年時点で入手可能なデータだけ**で再現しなければならない。未来の情報が混じると検証が無意味になる。

| 入力 | PIT再現可否 | 制約 |
|---|:--:|---|
| 株価 | ✅ | `price_history.date ≤ as_of`（2000〜） |
| 年間配当 | ✅ | `dividend_annual.fiscal_year ≤ as_of`（1989〜・深い） |
| 財務 | 🔶 | `financial_snapshots.fiscal_year ≤ as_of`。**EDINET 2008〜が下限**＝財務系指標のバックテストは2008以降のみ |
| result_overrides | ✅ | **`as_of_fiscal_year ≤ as_of` のoverrideのみ採用**（2025公表の連続年数を2015の検証に使わない）。既存設計のas_ofがPITを支える |
| beta | ✅ | as_of時点までの株価×指数で回帰 |

> **バックテストの深さは指標で異なる**: 配当系1989〜・価格系2000〜・財務系2008〜。これを混同せず、各検証に「有効開始年」を明示する（正直な検証＝定款）。

### 1.2 生存バイアス（survivorship bias）の排除
- 時点ユニバースは、**その時点で上場していた銘柄**（後に上場廃止した銘柄も含む）でなければならない。現役だけで検証すると「生き残った優良株」に偏り、減配・破綻リスクを過小評価する。
- 必要: `is_active`＋`delisting_date`（クラスC・要収集）。上場廃止銘柄の配当・株価も保持（少なくとも廃止前まで）。
- 廃止は重要アウトカム（減配の極端形）。**delisting_date収集をバックテスト前提条件に格上げ**する。

## 2. アウトカム定義（「望ましい結果」とは何か）

増配株戦略なので、アウトカムは**インカムの持続性と総合リターン**で測る。すべて as_of から前方（forward）。

| アウトカム | 定義 | 検証する主張 | 算出元 |
|---|---|---|---|
| **減配回避** | 前方N年（3/5）で**減配しなかったか**（binary/回数） | 配当claim・連続非減配・YoC質ゲート | dividend_annual（前方） |
| **増配実現** | 前方N年のDPS成長率・前方YoC | 配当claim・増配持続性 | dividend_annual・price |
| **トータルリターン** | 前方N年の株価騰落＋受取配当（再投資込み） | 総合スコア全体 | price＋dividend |
| **最大ドローダウン耐性** | 危機局面（コロナ等）での下落・減配 | 健全性claim・危機耐性フック | price・dividend |

> インカム戦略なので**減配回避が第一アウトカム**。総合リターンは補助（バリュエーション・成長claimの検証用）。

## 3. 評価メトリクス（モデルは効いているか）

### 3.1 全体
- **順位相関（Spearman）**: as_of のスコア順位 vs 前方アウトカム順位。指標/claim/総合の各レベルで算出。
- **分位スプレッド**: スコア上位分位 vs 下位分位の、前方減配率・前方リターンの差。**上位が下位より減配が少なく・リターンが高い**ことが最低条件。
- **グレード較正**: grade_dividend=A の銘柄群が、C/D群より**実際に減配が少ないか**。グレードが「効く」ことの実証。

### 3.2 指標別アトリビューション（どの指標が信号を持つか）
- 各指標の **IC（情報係数）**＝指標値 vs 前方アウトカムの相関。
- **信号の無い指標を炙り出す**: ICが恒常的にゼロ近傍の指標は重み低下/廃止候補（柱2の進化＝根拠ある統廃合）。
- 切り口①の検証: **YoC質ゲート（sound/payout_driven/cyclical）が前方減配を予測するか**を直接テスト（イクヨ/日本製鉄型を弾けているか）。

### 3.3 過学習の防止（透明性優先）
- **ウォークフォワード**: 訓練期間（例 2008-2016）でメトリクスを見て、**検証期間（2017-2024）で確認**。期間外で崩れる重みは採らない。
- **解釈可能性を優先**: 機械的な重み最適化でブラックボックス化しない。**単調で説明可能な関係**（「自己資本比率が高いほど減配が少ない」等）を重視。findexの製品価値は透明性。
- パラメータ探索は粗いグリッドに留め、桁での妥当性確認に使う（小数点以下の最適化はしない）。

## 4. データモデル追記（D3への差分）

PITスコアとアウトカムを保持する。現行 `dividend_scores`（現在採点）とは分けて、再現可能なバックテスト結果を持つ。

| テーブル | 役割 | 主なカラム |
|---|---|---|
| `backtest_runs` | 1回のバックテスト実行 | run_id, rule_version_id, as_of_grid（例 各年6月末）, universe_def, params_json, created_at |
| `backtest_scores` | **PIT再現スコア**（as_of別） | run_id, code, as_of_date, total_score, score_json, grade_*（D4と同形・ただしPIT入力で算出） |
| `backtest_outcomes` | 前方アウトカム | code, as_of_date, horizon_y, fwd_div_cut, fwd_dps_cagr, fwd_total_return, fwd_max_dd |
| `backtest_metrics` | 評価結果 | run_id, level（total/claim/indicator）, key, metric（spearman/IC/decile_spread/grade_calib）, value, sample_n |

> `backtest_scores` は **PIT入力**で算出する点が現行スコアと違う。スコアラ本体（評価層）は同一を使い、**入力を as_of でフィルタした派生ビュー**に差し替えて回す（一方向フロー維持）。

## 5. バックテスト・ワークフロー（D7への追記）

```
[backtest run]
1. as_of グリッドを決める（例: 2008-2024 の各年6月末）
2. 各 as_of で:
   a. 時点ユニバース確定（その時点で上場＝is_active or delisting_date>as_of）  # 生存バイアス排除
   b. PIT入力ビュー構築（price≤as_of / financial.fy≤as_of / dividend.fy≤as_of /
      result_overrides.as_of_fiscal_year≤as_of）                            # 先読み排除
   c. derive（status付き）→ score（rules.yaml の対象版）→ backtest_scores
3. 各 (code, as_of) に前方アウトカムを算出 → backtest_outcomes
   （前方N年の減配有無・DPS成長・トータルリターン・最大DD）
4. メトリクス算出 → backtest_metrics
   （Spearman / 指標別IC / 分位スプレッド / グレード較正）
5. レポート（HTML）: 効く指標/効かない指標・分位スプレッド図・グレード較正表
6. 重み較正の提案 → 次 rule_version（v5）。ウォークフォワードで検証期間が崩れないこと
```

- **計算量**: as_of 17時点 × 全銘柄 × 指標。フェッチは不要（DB内データのみ）＝レート制限の外。重いが一括バッチで回せる。
- **golden→コホート→全銘柄の段階**: まず golden+コホート38社で配線を検証（メトリクスが算出できるか）、その後フルで統計的有意性を見る。

## 7. 前提条件と実装フェーズへの引き渡し

**前提（これが無いとバックテストが歪む）**:
- `delisting_date` 収集（生存バイアス排除）＝クラスC・新たに前提条件へ格上げ
- 株価2000遡及・財務はEDINET 2008〜（深さの限界を明示）
- result_overrides の as_of 厳守（PIT合成）

**実装フェーズ**（[実装計画](../IMPLEMENTATION-PLAN.md)に新Phase）:
- Phase 5.5（score後・verify前後）に「バックテストPhase」を新設: PIT入力ビュー → backtest_scores → outcomes → metrics → v5重み提案
- まず golden+コホートで配線検証、次にフル
- 出力はHTMLレポート（自前サイトの一部）

→ **結論**: バックテストにより「データだけでなくモデルにも確証」を持たせる。重みは旧PJ継承の手づけから、**前方アウトカムで較正されたv5**へ進化する。透明性を犠牲にせず（解釈可能・過学習回避）。

## 8. 実装結果（2026-06-17・Phase5.5 全完了）

`findex backtest --what outcomes|pit|metrics|all`（CLI連携）で3段が揃った。
- 5.5-a outcomes（`outcomes.py`）= 884行（コホート35社×as_of 2008-2020×horizon 3/5）
- 5.5-b PITスコア（`pit.py`）= 442行（as_of別in-memory DBで評価層を再実行・look-ahead排除）
- 5.5-c **metrics（`metrics.py`・新規）= 34件**。純Python実装（spearman/三分位スプレッド/IC/grade_calib）。
  単体テスト7件（`test_backtest_metrics.py`）で符号・同順位・定数列・サンプル不足を担保。pytest 84 passed。

**メトリクス所見（h5・コホート生存者標本ゆえ方向性の確認に留める）**:
| レベル | 指標 | 値 | 解釈 |
|---|---|---|---|
| indicator IC | consecutive_no_cut_years × fwd_div_cut | **−0.22** | 連続非減配が長いほど将来減配が少ない＝**配当安全性テーゼを方向性で実証**（最強信号） |
| indicator IC | consecutive_dividend_growth_years × fwd_div_cut | −0.14 | 同上（増配継続） |
| indicator IC | yield_on_cost_5y × fwd_total_return | −0.18 | 高YoCほど前方リターン低＝バリュートラップ/平均回帰の古典 |
| total | total_score × fwd_total_return | spearman −0.21 / 三分位 −0.49 | **高スコア群が価格リターンで劣後**（ディフェンシブ妙味）＝v5重み較正の客観材料 |
| claim | grade_dividend 較正 | **NULL（検証不能）** | PIT時点は財務欠落で全銘柄grade B＝変動なしを正直に可視化 |

**確証主義の徹底**: PIT時点(2008-2020)は財務(2021+)が届かず score_json は配当指標のみ
（n_scored≈2-4）。よって本バックテストは**実質「配当シグナルの検証」**であり、財務系/バリュ系の
検証には有報の多年バックフィル（2008まで）が必要＝未収集（§7前提・宿題）。grade_calib が NULL に
なるのもこの帰結で、「動いていない」のではなく「PIT時点で区別する材料が無い」を value=None で
明示している。生存バイアス（廃止株が無料データに無い）も未解消＝統計的有意性ではなく配線と
方向性の確認段階（フル銘柄＋廃止株収集後に有意性へ）。
