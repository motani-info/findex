D2.6: 取得限界の整理 と 評価軸への影響

作成日: 2026-06-14 ステータス: 実測ベース完了 親: 02_5-feasibility-findings.md（D2.5実測）。本書はその結果を「①取れないものの整理」「②それにより機能しない評価軸」に構造化する。 動機: 旧PJの最大の過ちは「取れないデータがあることに気づけず、誤った数字をそのまま評価・投稿した」こと（花王 連続増配 実36年→誤26年）。本書はそれを設計レベルで根絶するための土台。

パート1: 取得限界の整理（マスターデータとして完全には取れないもの）

データを「取得可能性のクラス」で分類する。これが定款「足りない部分を常に把握できる」の実体。

クラスA: 完全取得可（現在値・ほぼ全銘柄）

スナップショットとして問題なく揃う。 - 識別: code / name / market / sector33（JPX 100%） - 会計メタ: 決算日 / 連結（EDINETコードリスト 全社） - 現在財務: 売上 / 営業利益 / 純利益 / 総資産 / 自己資本 / 現金 / 発行株数（J-Quants /fins/summary・EDINET。実測カバレッジ95〜99%） - 現在の予想配当・配当性向（J-Quants 99.8%）

クラスB: 時系列に「下限」がある（過去に遡れない）

データ 下限 2000年以前 性質
年間配当 dps_annual FY1989（backfill） ✅一部 backfillした銘柄のみ。未backfillは2000-2002で頭打ち
日次株価 2000-01-04（yfinance） ❌ pre-2000は無料で存在しない（構造的）
財務諸表 2008（EDINET XBRL） ❌ XBRL制度開始。pre-2008は個別IR/有報PDFのみ
歴史的PER/PBR 実質2008 ❌ 株価×歴史的財務の重なり。pre-2008は算出不能

クラスC: フィールド自体が欠落（収集・ソース問題＝修正可能）

「取れないのではなく、取っていなかった／別ソースが要る」もの。

フィールド 現状 是正策
listing_date 0% kabutan等スクレイピング
edinet_code 0% EDINETコードリスト（即3,842件）
capital_expenditures 0% EDINET CF計算書
investment_securities 不在 EDINET 有報XBRL
深い株価（2000〜現在） 2024-06〜のみ yfinanceで2000年まで再取得

クラスD: 構造的に取得不能（諦めて「正直に扱う」対象）

  • 2000年以前の日次株価（→ pre-2000のPER/PBR・長期チャート不能）
  • 2008年以前の詳細財務（→ pre-2008の財務系指標の遡及不能）
  • 真の連続増配/非減配年数（backfill範囲外のpre-2000分） → 打ち切り＝N+でしか表せない

重要: クラスCは「直す」、クラスB/Dは「正直に扱う（評価不能・N+と明示し捏造しない）」。この区別が設計判断の核。

パート2: 揃わないデータで「機能しない評価軸」の影響調査

旧DB computed_metrics（3,746銘柄）で各指標の実効値率を実測。「真の欠損」と「正当な0（増配せず/減配あり）」を分離して、破綻の正体を特定した。

2.2 破綻タイプ別の整理

① field欠落で機能不全（修正可能・最優先） - FCF配当カバレッジ（真NULL 57%）: 原因は capex 0%。FCF=営業CF−capex が計算できず半数以上が欠損。→ EDINET CFでcapex取得すれば回復。 - ROIC-WACC（24%）/ ネットキャッシュPER（12%）: investment_securities不在・複合計算。→ EDINETで構成要素を埋めれば改善。 - 性質: これらは「データが無い」ではなく「取っていなかった」。クラスCの是正で直る。

② 履歴長不足で機能不全（一部は構造的） - 10年増配CAGR（35.6%）/ EPS成長5y（31.2%）: 11年/5年の履歴が要る。 - 若い銘柄（上場が新しい）はそもそも歴史が無い＝構造的に算出不能（クラスD的）。これは欠損でなく「評価対象外」が正しい。 - 既存銘柄でも配当/財務の遡及下限（配当2000floor・財務2008）に当たると不足。 - 性質: 「データを増やせば直る部分」と「銘柄が若くて原理的に無理な部分」が混在。後者をNULL扱いで減点するのは誤り（D4 Nullポリシーで是正）。

③ 打ち切りで「静かに誤る」（最危険・前回の過ちの本体） - 連続非減配/増配/減配信頼性は、配当履歴の下限で左打ち切りされる。 - streak_is_censored=1 が 1,039銘柄（27.7%）。これらは「16年」のような確定値に見える過小な数字を出す（花王26 vs 36の構造）。 - NULLでないので品質フィルタをすり抜けるのが致命的。値はあるが間違っている。 - 性質: backfill範囲外のpre-2000はクラスD（構造的）。N+表示（「N年以上」）でしか正直に表せない。listing_dateで「真の新規上場」と「打ち切り」を区別してN+を減らす。

④ 影響軽微（現在値スナップショットのみ使用） - 自己資本比率・ROE・営業利益率・売上5yCAGR・予想配当性向・ミックス係数・配当利回り。 - これらは現在値で完結するため、価格2000年・財務2008年の下限の影響をほぼ受けない。実効値率90%以上。

2.3 重要な構造的洞察

findexの核心（最高重み）が、最も取得が難しいデータ領域に乗っている。

  • 配当継続性（連続非減配2.5＋連続増配1.5＋減配信頼性0.8＋10年CAGR1.2 = 重み6.0、全体の約1/3）は、すべて深い配当履歴に依存し、打ち切り（27.7%）の直撃を受ける。
  • 一方、健全性・割安度（現在値系）は安定して取れる。
  • つまり「増配株分析」というfindexのアイデンティティは、まさにデータが最も不確かな場所に成立している。だからこそ ①N+の正直運用 ②backfill ③listing_dateでの打切り判定 が品質の生命線。

2.4 価格2000・財務2008の下限は「現在スコアリング」には実はほぼ効かない

  • 価格2000下限: 配当利回り・PER/PBR系は現在値を使うため、現在のスコアリングには影響しない。効くのはバックテストと長期チャート投稿（D5）。
  • 財務2008下限: 5年成長は直近5年（2021-26）で足り、現在のスコアリングには影響しない。効くのは長期の時系列分析。
  • → 下限B/Dの実害は「現在スコア」より「過去再現（バックテスト）と長期フック投稿」に出る。現在スコアの実害の主因は ① capex欠落 と ③ 打ち切り。
