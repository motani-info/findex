D4.5: 指標較正（v3 → v4提案）

作成日: 2026-06-14 親: D4 指標システム / 分析切り口① / D2.6影響 目的: D4で洗い出した指標設計そのものの歪みを、実データに基づいて較正する。設計のみ（実装凍結）。確定すれば次の rule_version（v4）として版管理。 方針: X発信(D5)より較正を優先（ユーザー指示）。閾値は旧DB実測の分布で裏づける。 ステータス: ✅ v4確定（2026-06-14・§7の全項目ユーザー承認）

0. 較正対象サマリ

# 問題（D4で検出） 較正
① 10年CAGRは低ベース/一過性を過大評価 YoC（取得利回り）に置換＋EPS牽引ゲート＋DOE追加
② 自己資本比率80%満点は上位10%のみ・ROEと数学的に矛盾 閾値70%へ・ROE再較正・矛盾を緩和
③ 営業利益率の絶対20%が低マージン業種を不当に0点 業種相対スコア導入
④ 予想配当性向・利回りが予想値単一依存 実績フォールバック＋confidence
⑤ 成長の三重カウント（EPS・売上・配当CAGR） ①でCAGR除去により自然に解消

1. 較正①: 10年CAGR → YoC ＋ 持続性ゲート ＋ DOE

切り口①（analysis-angles.md）の結論を指標に落とす。

変更

  • 廃止: dividend_growth_10y_cagr（weight 1.2）。理由=低ベース/市況回復を過大評価（イクヨ66x・日本製鉄36x が上位）。
  • 新設: yield_on_cost_5y（主）＝最新DPS ÷ 5年前株価。低ベースノイズを自動抑制。weight 1.2 を継承。
  • 補助に yield_on_cost_3y、dividend_multiple_5y（dividend_annualのみで算出可・株価不要）。
  • データ要件: 5年前株価（株価2000遡及が前提・クラスC）。それまでは dividend_multiple_5y を暫定代理。
  • 持続性ゲート（質）: YoCの生スコアに増配の質を掛ける。分解式 DPS倍率 = EPS倍率 × 配当性向変化 から判定:
  • sound（EPS牽引・EPS倍率≥1.5）→ ×1.0
  • payout_driven（性向拡大依存）→ ×0.5
  • cyclical（EPS横ばい/減・赤字）→ ×0.3
  • これにより「高YoCだが一過性」を自動減点（イクヨ/日本製鉄を弾く）。新指標を増やさずYoCの質補正で実現＝既存の配当性向/FCF/EPS指標と二重計上しない。
  • 新設候補: doe（=年間配当総額÷自己資本＝ROE×配当性向）。weight 0.8。利益が振れても自己資本は安定→配当性向より持続性シグナルとして優秀。日本で配当政策指標として普及。

2. 較正②: 自己資本比率の閾値ダウン＋ROE矛盾の緩和

実測（旧DB）

  • 自己資本比率: 中央値54.9%、75%ile 70.1%、90%ile 80.2%。現閾値80%＝上位10%のみ満点＝厳しすぎ。
  • ROE: 中央値8.3%、75%ile 13.3%、90%ile 20.8%。現閾値20%＝上位10%。
  • 矛盾実測: 自己資本比率80%+かつROE20%+は16社（0.4%）＝両立ほぼ不能（分母の資本が増えるとROEは下がる数学的関係）。

較正

  • 自己資本比率: 満点閾値 80% → 70%（上位約25%）。安全性は方向として高いほど良いが、80%超は過小レバレッジ＝資本非効率の領域なので、満点を70%に置き「健全な厚み」を評価。weight 1.5 維持。
  • （任意）>90% かつ ROE低 は資本非効率として軽い頭打ち＝将来検討。
  • ROE: 満点閾値 20% → 15%（上位約25%）。20%は稀すぎ。weight 1.5 維持。
  • 矛盾の扱い: 両者は「安全性（資本の厚み）」と「効率性（資本の稼ぐ力）」の意図的なペア。閾値を現実水準（70%/15%）に下げることで、優良企業が両方で適正得点を取れるようにし、ゼロサムの綱引きを緩和する。

3. 較正③: 業種相対スコアの導入

実測（営業利益率の業種別中央値）

卸売3.3%・小売3.7%・食料品3.4% … vs 銀行19.5%・証券19.8%。絶対20%満点では低マージン業種はベスト企業でも常に0点。

較正

  • operating_margin を 業種相対スコアに変更: sector33内のパーセンタイル順位 → 0〜10点（業種内で上位ほど高得点）。
  • 「卸売業の中で営業利益率トップ」は高評価されるべき＝業種の構造差を吸収。
  • 暴走防止に絶対フロア併用（業種全体が赤字水準なら相対上位でも満点にしない）。
  • 候補拡大: roe も業種相対の余地（金融は別途swap済み）。まず営業利益率で導入し効果を見て判断。
  • 据え置き（絶対のまま）: 自己資本比率（安全性は絶対量に意味）、配当利回り・配当性向（投資家視点の絶対水準）。

4. 較正④: 予想値依存のフォールバック

payout_ratio（weight 2.0・最高位級）と div_yield は forecast_dps（会社予想）依存。予想欠損・保守性で歪む。

較正

  • 予想欠損時は実績TTMの年間配当にフォールバックし、confidence=present（予想ありは verified 寄り）を立てる。
  • Nullポリシー（D4）と接続: forecast欠損→missingで分母除外、ではなく実績で代替して採点継続（高ウェイト指標を簡単に落とさない）。代替したことは status/confidence に記録。

5. 較正⑤: 成長軸の整理

  • ①で10年CAGRを除去 → 成長軸は eps_growth_5y と revenue_growth_5y_cagr の2本に。
  • インカム側（連続増配・YoC・DOE・配当性向・利回り）とのバランスが取れ、成長三重カウントは解消。
  • 追加変更なし（①の副次効果として完了）。

7. 確定事項（ユーザー承認 2026-06-14）

  1. YoCの質ゲート＝掛け算で減点（×1.0 sound / ×0.5 payout_driven / ×0.3 cyclical）。厳格除外ではなく段階評価で採用。✅

  3. 自己資本比率 満点閾値 70%、ROE 満点閾値 15% に較正。✅
  4. 業種相対は営業利益率のみ先行（効果を見てROEへ拡大判断）。✅（既定）
  5. YoC暫定運用: 株価2000遡及までは dividend_multiple_5y を代理に使う。✅（既定）

→ v4として確定。rule_versions 設計に反映。実装フェーズで rules.yaml を更新し（null_policy: status_based・YoC質ゲート・DOE・閾値・業種相対）再スコア。capex/investment_securities/cost_of_debt/株価2000遡及の取得後に #6FCF・#12ROIC・YoC本算出を有効化。

8. v4は閾値較正どまり・重みはv5でバックテスト較正（design-review #1）

v4で較正したのは閾値（70%/15%・業種相対）と指標構成（YoC/DOE/CAGR廃止）であり、重み配分（2.5/1.5/1.2/0.8…）は旧PJ継承の手づけのまま。これらが前方アウトカムを予測するかは未検証。 - 重みの妥当性は D8 バックテストで検証: 指標別IC（情報係数）・分位スプレッド・グレード較正で「効く指標/効かない指標」を炙り出し、客観的根拠で重みを改訂 → v5。 - v4の質ゲート（YoC×sound/payout_driven/cyclical）も前方減配を予測できているかをD8で直接検証（イクヨ/日本製鉄型を実際に弾けているか）。 - 過学習回避（ウォークフォワード）・解釈可能性優先は D8 の原則に従う。v5は「手づけ」から「アウトカム較正済み」への進化。
