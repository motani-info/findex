設計レビューと是正（フラット第三者レビュー）

作成日: 2026-06-15 位置づけ: 設計フェーズ完結（D1〜D7・D5）後に、実装着手前へ向けてフラットな第三者視点で全設計を点検した記録と、各指摘への是正方針・反映先。本書は是正の索引であり、実体は各設計書の改訂とD8新設に落ちる。 動機: 設計品質が今後の製品品質を直接決める。「データだけ厳密でモデル・プロダクトが手薄」という重心の偏りを実装前に正す。

0. 総評

観点 評価
データ基盤（柱1）の厳密さ 高い。自己認識・5状態Null・claim別グレード・N+・結果補正・来歴メタ・D2.5実証は多くの製品より堅い
分析モデル（柱2）の妥当性 盲点。重みは旧PJ継承の手づけ、スコアが何かを予測する実証が無い
プロダクト/読者（柱3） 手薄。読者像・競合・差別化テーゼ・出力物の実体設計が弱い

→ 重心が「入力の正確性」に偏り、定款が警戒した完璧主義が投資配分そのものに出ていた。本レビューでモデルとプロダクトの設計を引き上げる。

2. 是正の中身（要点）

2.1 モデル検証＝D8バックテスト基盤（#1・最重要）

  • PIT（Point-in-Time）原則: as-of 時点で入手可能なデータだけでスコアを再現（先読みバイアス排除）。result_overrides.as_of_fiscal_year が既にPIT合成を支える設計になっている点を活用。
  • 生存バイアス排除: 上場廃止銘柄（delisting_date）を時点ユニバースに含める。
  • アウトカム: 前方N年の減配有無・増配継続・トータルリターン。
  • 評価: スコア×アウトカムの順位相関・分位スプレッド・指標別IC・グレード較正（grade-A配当claimは本当に減配が少ないか）。
  • 重み較正: ウォークフォワード（訓練/検証分割）で過学習を防ぎ、解釈可能性を優先（ブラックボックス化しない＝透明性が製品価値）。
  • 詳細は D8 バックテスト基盤。

2.2 プロダクト/読者（#2・#6）

  • 差別化テーゼ: 「増配率の高さに騙されない、質（EPS牽引）で選ぶ増配株」＝切り口①を製品の中心に。汎用ランキングは誰でも出せる。
  • 出力チャネル: Xメイン。ただし投稿画像（ランキング表・チャート）はローカルHTMLで生成＝自前サイトを兼ねる。サイトは確実に動きSEO資産にもなる。Xの脆い自動化は手動承認フォールバック付き。
  • コンプラ: 個別銘柄の売買推奨はしない／免責定型／シミュレーションは前提明示。

2.3 データ品質の検証強化（#3・#7・#4）

  • 株価履歴（yfinance 2000-2024）: golden照合源が無い区間なので、複数ソース突合・分割イベント独立検証・外れ値検知でYoCの信頼度を担保（無理なら信頼度に上限を付け正直に）。
  • 旧DB配当の能動洗浄: yfinance配当1999+の再取得と移行データの相互照合。
  • EDINETリスク低減: 削らず、パース成功率をゲート化し基準別辞書を独立検証（golden_financialsで基準を散らす＝D6既設計を前倒し運用）。

2.4 整合性（#5・#8・#9）

  • ランキングはclaim内が主、総合は足切り付き参考値。
  • ユニバース定義を確定（普通株のみ・約数）。
  • D3を単一正本化（themes/post_queue/post_log）。運用設計を実装計画へ。
